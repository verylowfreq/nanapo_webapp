# nanapo_webapp

[ビット・トレード・ワン](https://bit-trade-one.co.jp/)社のUSB 7セグメントディスプレイ「[Nanapo (AD7SGPR)](https://bit-trade-one.co.jp/ad7sgpr/)」をブラウザから操作するWebアプリです。ビルド不要の静的サイトで、GitHub Pages（`/docs`）でそのまま公開できます。

## できること

- PCブラウザ・Androidブラウザから、USBで接続したNanapoへテキストボックスの文字列を送信し、7セグディスプレイに表示させる
- 「スクロール表示」「通常表示」「時計表示」を選択可能
  - スクロール表示（最大40文字）: 右から登場し、8文字以内に収まる場合は右詰めで一時静止してから左へ退場、8文字を超える場合はそのまま左へ流れ続けるマーキー表示になる。いずれも停止するまで無限に繰り返す
  - 通常表示（最大8文字）: アニメーションなしで即座にそのまま表示する
  - 時計表示: `12-34 .56`（時-分 .秒）の形式で現在時刻（24時間表示）を毎秒更新し続ける。停止するまで繰り返す
- `@CLR` / `@ANI` / `@7SG` / `@SGR` / `@BRI` / `@BRR` / `@HEX` など各種コマンドをボタンひとつで送信する
- Nanapo本体のボタンA（輝度）・B（7セグ表示ON/OFF）・C・Dの操作状態をリアルタイムに表示する（各表示は3秒で自動的にクリアされる）

## 対応環境

- **PC**: Google Chrome / Microsoft Edge などのChromium系ブラウザ（[Web Serial API](https://developer.mozilla.org/en-US/docs/Web/API/Web_Serial_API) にネイティブ対応）
- **Android**: Chrome（[WebUSB API](https://developer.mozilla.org/en-US/docs/Web/API/WebUSB_API) 経由。Android版Chromeの Web Serial API はBluetoothシリアルのみの部分対応でUSBシリアルには使えないため、Android端末を検出した場合は常に [google/web-serial-polyfill](https://github.com/google/web-serial-polyfill) でWebUSB上にSerial互換のAPIを実装して利用します）
- HTTPS（またはlocalhost）での配信が必須です。GitHub Pagesはこれを満たします。
- Android端末の機種・OSによっては、標準のUSBシリアルドライバが先にデバイスを占有し、WebUSB経由で掴めない場合があります。その場合はブラウザからの接続に失敗します。
- ページを開くと「受信ログ」に `[判定]` で始まる行が出力され、Web Serial APIとWebUSB（ポリフィル）のどちらが使われているかを確認できます。想定と異なるAPIが使われている場合はこのログを確認してください。

## 使い方

1. USBケーブルでNanapoをPCまたはAndroid端末に接続する
2. 公開されたページを開き、「接続」ボタンを押してデバイス選択ダイアログからNanapoを選ぶ
3. 表示モードを選ぶ
   - 「スクロール表示」（最大40文字）か「通常表示」（最大8文字）の場合は、テキストボックスに表示したい文字列を入力して「送信」ボタン（またはEnterキー）で送信する
   - 「時計表示」の場合はテキストボックスは使用せず、「送信」ボタンを押すと現在時刻の表示が始まる
   - スクロール表示・時計表示はどちらも停止するまで無限に繰り返す。「停止」ボタンを押すか、別の文字列・コマンドを送信すると、その時点で打ち切られる（停止ボタンは表示を`@CLR`でクリアする）
4. 「各種コマンド」パネルのボタンを押すと、対応するコマンドがその場で送信される（`@HEX`は16桁の16進数を入力してから送信）
5. 「本体ボタン状態」パネルには、Nanapo本体のボタンA（輝度）・B（7セグ表示ON/OFF）・C・Dを押したときの状態がリアルタイムに反映される。各表示は3秒後に自動的にクリアされる

## 通信仕様

Nanapo (AD7SGPR) 側の仕様に合わせて、以下の固定パラメータで通信しています。

- 通信速度: 115200bps
- 改行コード: CRLF (`\r\n`)
- Nanapo自体は一度に最大8文字（8桁）までしか表示できません。スクロール表示はこの制約を踏まえ、入力文字列の前後に8文字分の空白を付加してから8文字幅の窓を1文字ずつスライドさせ、生成した各フレームを約250msごとに送信することで実現しています（8文字以内の文字列は右詰めの状態で約1.5秒静止してから退場し、これを停止するまで繰り返します）
- 通常表示は入力文字列をそのまま1回だけ送信します（アニメーションなし）
- ピリオド「.」は実機では独立した桁を占有せず、直前の文字の小数点として表示されます。そのためアプリ内の文字数カウント（8文字/40文字の上限判定や、スクロールの右詰め位置の計算）ではピリオドを1文字として数えません（例: `1.2.3.4.5.6.7.8`は15文字ですが8桁として扱われ、通常表示でもそのまま送信できます）

詳細は [Bit Trade One 公式ドキュメント（AD7SGPR）](https://github.com/bit-trade-one/AD7SGPR) の `SendCommandSpecification.md` を参照してください。

## ローカルでの動作確認

`docs` ディレクトリを任意の静的サーバーで配信してください。

```sh
cd docs
python3 -m http.server 8080
```

`http://localhost:8080/` をChromeで開いて確認できます。
